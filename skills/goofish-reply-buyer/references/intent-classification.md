# 买家意图分类

Agent 读完 `message_history` 后要做的第一步判断。分 5 大类 + 常见子类。

## 类 1 · 询价 / 信息问询（低摩擦，直接回）

### 子类 1.1 "还在吗 / 在不在"
- 典型：单独一句"在吗"、"在不在"、"还在吗"
- 心态：用户刚点进来，想确认卖家是活人
- 回复模板：`在的` / `在~` / `在的，有需要可以直接问`

### 子类 1.2 "多少钱"
- 典型：挂价很明显但对方还问
- 原因：对方可能在看商品列表批量问（为后续议价铺垫）
- 回复模板：`X 元哦，详情写在描述里了` + 可选议价暗示

### 子类 1.3 功能 / 规格问询
- 典型："支持 5G 吗"、"M 码胸围多少"、"续航多久"
- 原因：描述里没写清楚或用户没看完
- 回复：回答 + 建议"详情可以看图 X"

### 子类 1.4 物流问询
- 典型："能发 XX 吗"、"多久到"、"包邮吗"
- 回复：回答 + 明确时效

## 类 2 · 议价（高频）

详见 `bargain-ladder.md`。3 档子类：

### 子类 2.1 小刀（5-15%）
- 典型："1500 能便宜点吗"（原价 1600）
- 策略：**友好接**，可以让 5-10%

### 子类 2.2 大刀（15-40%）
- 典型："1200 出不"（原价 1600）
- 策略：谨慎回，看商品是不是好卖；不是就委婉拒或给中间价

### 子类 2.3 屠龙刀（>40%）
- 典型："500 卖给我"（原价 1600）
- 策略：礼貌拒，不解释太多（容易被纠缠）
- 模板：`谢谢啦，这个价没法卖，您再看看其他的`

### 子类 2.4 无差别砍价（没出具体价）
- 典型："便宜点"、"最低多少"
- 策略：让对方先出价
- 模板：`您出个价我看看～` / `看您诚意～方便说个数吗`

## 类 3 · 催付 / 订单跟进

### 子类 3.1 买家已付款问发货
- 典型："付款了明天发吗"
- 回复：确认时间 + 发货后给单号

### 子类 3.2 买家已下单未付款（卖家想催）
- 这个是**卖家主动发起**的场景
- 策略：温和，不要压力
- 模板：`亲亲有准备付款吗，在的话我今天打包好明早寄出哦～`
- 禁忌：不要多次催（> 2 次 = 骚扰）

### 子类 3.3 已发货但买家问"到哪了"
- 卖家看单号同步一下
- 模板：`刚查了在 XX 中转，预计明天上午` + 附单号

## 类 4 · 售后 / 纠纷（高敏感）

### 子类 4.1 "到货有问题"
- 先别急着承认 / 否认，先问细节 + 让对方拍图
- 模板：`方便拍几张图看下吗，图里看不太清楚具体是哪里不对`
- **同时**让卖家本人介入判断

### 子类 4.2 "申请仅退款"
- **立即通知卖家本人**——Agent 不替拒绝也不替同意
- 起草"请申请退货退款（寄回后退钱），不接受仅退款"的话术
- 同时提醒卖家留聊天记录证据

### 子类 4.3 "要差评你信不"
- 这是**威胁**——别上套
- 话术模板：`商品是按描述卖的，您可以按平台规则申请售后，有问题平台会介入处理哦`
- 不要妥协给钱（助长黑产），不要撕（升级损失）

### 子类 4.4 "用了几天坏了 / 和宣传不一样"
- 先判断是不是卖家过错
- 描述里已经标注的瑕疵 → 有底气拒
- 没标注的瑕疵 → 建议卖家酌情补偿（比如退部分钱）

## 类 5 · 风险 / 合规红线

### 子类 5.1 外联请求
- "加个微信聊"、"VX 多少"、"私聊"
- **必须** block + 合规话术替换（见 `goofish-risk-guard/references/external-contact-keywords.md`）

### 子类 5.2 诱导线下担保
- "先转 50 定金支付宝"、"直接转账便宜点"
- 合规替换："不好意思哈，交易还是走闲鱼担保"

### 子类 5.3 诈骗模式
- "我是客服你账户异常"、"要验证码"、"点个链接"
- 模板：`这看起来像诈骗，我不会按这个操作，您如果是真买家请直接下单` + 建议卖家拉黑
- **告诉卖家**：点链接 / 给验证码 = 账号可能被盗

### 子类 5.4 骚扰 / 情绪攻击
- 连续性辱骂、纠缠
- 建议卖家拉黑 + 举报
- Agent 不帮写反击话术

## 不确定怎么办

- **信息不足** → 拉更多 history（从 20 条扩到 50 条）
- **跨意图**（议价 + 外联）→ 优先处理风险意图（外联先 block）
- **纯表情 / 无意义** → 回问开放问题："方便说一下您想了解哪方面呢"

## 分类用 LLM 判，不用写规则

本文件是**知识**不是**代码**。Agent 看完这里就能自行判定。不用在 skill 里写硬编码规则引擎——LLM 在上下文里做意图分类更准。
