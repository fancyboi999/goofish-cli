# 流量掉 / 商品没曝光的归因清单

按**可能性从高到低**排。Agent 诊断时从头往下过。

## 归因 1 · 错过 24 小时初始流量池

**症状**：发布后第一天曝光还行，之后掉成 0
**原因**：初始推荐窗口 24 小时内互动数据（点击/收藏/咨询）不够 → 不进下一级池
**证据**：item_view 显示发布 > 24h，曝光集中在第 1 天
**修复**：
- **不要**删除重发（会被标记为同款复活）
- 可以编辑标题（24h 后）尝试新关键词
- 同时在描述里补细节、首图换更吸睛的
**评级**：高频，约 40% 流量掉由此引起

## 归因 2 · 降价改标 → 虚假促销惩罚

**症状**：初期曝光正常，改价后突然掉
**原因**：系统检测"先降价再改标题"的组合 → 判定虚假促销 → 搜索权重 -50%~70%
**证据**：
- item_view 的价格有变更记录
- 降价后 24h 内标题也改了
**修复**：
- 静待 7 天权重恢复
- 期间不要再改任何内容
**评级**：极高，约 25% 限流由此引起

## 归因 3 · 类目错放

**症状**：搜核心词找不到商品；搜更精准的长尾词能找到
**原因**：类目和商品实际类型不匹配 → 权重打折或进审核
**证据**：
- 当前 catName 和 `category_recommend(title=当前标题)` 返回的不一致
- 同类热销商品（搜出来的前 5 条）的类目更合理
**修复**：
- 重新发布（删旧发新，而不是在原商品上改类目 — 改类目等于重新审核）
- 注意**删旧时机**：不要短时间频繁删发（职业卖家嫌疑）
**评级**：中频，约 15%

## 归因 4 · 标题没 SEO 优化

**症状**：搜"通用词"（如"耳机"）找不到，搜"完整标题"能找到
**原因**：
- 标题缺品牌、缺核心词
- 关键词没前置
- 冷僻词多，热门词少
**证据**：item_view 的标题 vs `goofish-publish-item/references/title-formula.md` 的 5 段式
**修复**：按 5 段式重写标题（24h 后改一次就好，不要频繁）
**评级**：高频，约 20%

## 归因 5 · 图片不合规 / 非 1:1 首图

**症状**：搜索排名不高，点击率低
**原因**：
- 首图非 1:1 → 推荐算法扣分
- 图片数量少（< 3 张）→ 信任感低 → 点击率低 → 正反馈消失
**证据**：item_view 返回的 `images[0].width != height` 或 `len(images) < 3`
**修复**：补图（尤其首图换 1:1 实拍）
**评级**：中频

## 归因 6 · 外联词历史 / 违规记录

**症状**：整个账号的所有商品都没流量（不只一件）
**原因**：账号本身被降权（闲气值 < 80）
**证据**：
- `auth_status` 返回 OK 但多件商品都曝光惨
- 用户提到最近有过违规记录（被投诉、删除过消息）
**修复**：
- 本工具无法解，建议用户在 App 看"违规记录"
- 一段时间内（7-30 天）慎发、慎回复
**评级**：中频

## 归因 7 · 价格明显不合市场

**症状**：曝光有但咨询 0
**原因**：
- 定价过高 → 买家看一眼就走
- 定价异常低 → 买家怀疑有坑（钓鱼/诈骗）
**证据**：
- search_items 同类前 5 条的价格区间 vs 当前挂价
**修复**：调整到市场 80% 分位左右（可以略高于平均但不要高太多）
**评级**：中频

## 归因 8 · 发布频次 / 账号新

**症状**：新号或刚复活的号，所有商品都没流量
**原因**：新号权重低，实名认证没做
**证据**：用户说"账号刚注册"、"最近闲鱼没登录过"
**修复**：
- 做实名认证
- 降低发布频次（新号前 3 天 ≤ 3 件/天）
- 先用账号买点小东西（增强账号活跃度）
**评级**：低频但极难纠正

## 归因 9 · 多号同 IP 关联

**症状**：用户用家里 WiFi 注册了多个号，其中一个违规 → 全部被限流
**原因**：同 IP 多号触发关联风控
**证据**：用户承认有多个号在同 IP 操作
**修复**：
- 本工具当前是单 session，不做多号运营
- 建议用户：不要在同家 WiFi 操作多号
**评级**：低频但难纠

## 归因 10 · 类目本身冷

**症状**：商品没问题，但曝光就是少
**原因**：所选类目日活低（比如特别小众的手办、古董）
**修复**：
- 这是**结构性问题**，没法靠优化解决
- 考虑拓展到相邻热门类目（加搜索词）
**评级**：低频

## 归因综合使用

Agent 同时打多个分，给出**前 3 个最可能**，不要只给一个：
- 如果发现有降价改标 + 类目错放 → 两个都列，先处理降价改标（时间会帮你）
- 如果降价改标 + 标题差 → 两个都修，但不要同一天改

## 观察周期

修完之后观察多久？
- **小修**（改图、补描述）→ 24-48h 看新曝光
- **大修**（重发、改类目）→ 需等 24h 初始窗口 + 3 天看稳态
- 修完立刻看没变化 = 正常，别再动

## 告诉用户"限流不是绝症"

很多卖家一看"限流"就慌，想着"刷单救回来"。Agent 要传递正确认知：
- 限流是**阶段性**的（多数 7-30 天）
- 不要再操作加深风险
- 刷数据 / 换号 都是更糟的选择
- 最好的恢复 = 时间 + 按规则发新商品
