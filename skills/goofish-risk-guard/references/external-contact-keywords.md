# 外联词（消息 + 描述双重红线）

**这是本工具合规硬红线之一**。Agent 检测到用户草稿里含外联词，必须：
1. 阻止发送
2. 告诉用户命中了什么词
3. 提供合规替换话术

## 核心关键词表

### 平台类
| 原词 | 常见变体 |
|---|---|
| 微信 | 威信 / 葳信 / 维信 / 围信 / 维❤ / 微❤ / 薇信 / 媺信 / 微X / 溦信 |
| V信 / VX | v信 / V❤ / v❤ / Vx / vX / VX信 |
| QQ | 企鹅号 / 扣扣 / 企鹅 / Q号 |
| 电话 | 手机号 / 座机 / 移动号 / 联通号 / 打电话 |
| 钉钉 / 飞书 / 探探 / Telegram / TG / 小飞机 |

### 诱导动作类
- 加我 / 加一下 / 加我好友 / 加到 / 加 XX
- +我 / ➕我 / ﹢我 / 加v / 加VX
- 扣我 / 扣 XXX / Q我
- 私聊 / 私信 / 走X / 走别的
- 留号 / 留联系 / 留方式 / 发号 / 发我
- 搜我 / 搜索 XXX

### 身份类
- 老板 / 哥 / 姐 + 加 + 平台名
- "方便聊" / "方便说" + "加 XX"

### 数字变形（正则检测）
- 11 位手机号：`1[3-9]\d{9}`
- QQ 号：`[1-9]\d{4,11}` + 上下文含"Q"
- 微信 ID：含字母+数字+下划线，上下文含"加"/"微"

### 图像 / 二维码
- 买家说"我发二维码给你" / "扫我" → 警告
- 买家发图时 Agent 无法自动扫，但要提示卖家"别扫二维码，保持闲鱼内沟通"

## 合规替换话术库

### 买家问"加个微信"
**原话（禁）**：好的，我微信是 xxxxx
**合规替换**：抱歉哈，为了保证交易保障，咱们在闲鱼里聊就行，有啥问题尽量详细发给我～

### 买家问"能便宜点走支付宝吗"
**原话（禁）**：可以，支付宝转我 xxx
**合规替换**：价格咱们可以聊，但付款还是走闲鱼担保交易哈，这样对你对我都有保障～

### 买家说"我微信：xxx 加我"
**原话（禁）**：好的我加你
**合规替换**：不好意思哈，闲鱼规则不让外联，有事在这里问我就行

### 卖家想留联系方式（商品描述）
**原话（禁）**：详询加 VX：xxx
**合规替换**：详情可以在闲鱼私信我，看到就秒回

### 卖家想给买家发自己的闲鱼主页
**原话（禁）**：去我微信看别的货
**合规替换**：主页上有更多宝贝，欢迎看看～

## 扫描顺序（Agent 实操）

1. **精确匹配**：上表所有"原词"扫一遍
2. **变体正则**：对"微信/VX/威信/V信"系列用正则覆盖拼音、谐音、emoji 组合
3. **号码提取**：手机号 / QQ 号正则
4. **上下文判定**：如果草稿里有独立数字串，**配合**"加"/"联系"/"留"等动词，升级为 block

```
命中 → block=True，reason="外联词: {word}"，replacement={上表合规话术 或 通用模板}
```

## 灰色区

有些词本身不违规，但组合起来才违规：

| 单看 OK | 组合触发 |
|---|---|
| "方便" | "方便加我微信吗" |
| "直接" | "直接走转账" |
| "详情" | "详情加我" |
| "联系" | "手机联系" |

Agent 用 LLM 判断整句语义比硬扫词表更准，但词表作为兜底。

## 为什么外联红线这么硬

1. **平台侧**：闲鱼担保交易是核心商业模式，线下交易绕开平台手续费 → 平台零容忍
2. **用户侧**：线下交易后出纠纷平台不介入 → 买家被骗、卖家挨骂，都回头骂平台
3. **法律侧**：闲鱼担保基于支付宝，线下付款规避了反洗钱监测 → 涉合规

**踩线后果**：轻则被限流 30 天、闲气值 -10，重则封号（永久）。
