# 图片检查清单

## 硬性规则（不符合就阻止发布）

- [ ] **总图数 ≤ 9**
- [ ] **首图 1:1 正方形**（不是就让用户换）
- [ ] **不能是网图 / 盗图**（买家反向搜图能查到平台下架）
- [ ] **不能含联系方式**（二维码、手机号、微信号在图上都算外联）
- [ ] **不能有第三方平台水印**（淘宝链接、小红书账号、抖音号等）
- [ ] **不能含广告法违禁内容**（绝对化词做成图、医疗夸大宣传图）

## 软性建议（影响曝光和买家信任）

- [ ] 图 ≥ 3 张，< 3 张提示用户补
- [ ] 至少 1 张实拍细节图（证明不是网图）
- [ ] 有瑕疵 → **单独 1 张瑕疵特写**（这是仅退款纠纷时的救命证据）
- [ ] 建议按"整体→细节→瑕疵→场景"顺序排
- [ ] 光线自然，背景干净（浅色桌面、白墙，不要乱成一团）

## 图片顺序建议

```
图 1：首图（1:1，整体外观正面）
图 2：侧面 / 背面
图 3：重要细节（标签、型号、做工）
图 4：功能演示 / 使用场景
图 5：瑕疵特写（如有）
图 6-9：原包装 / 配件 / 购物凭证（小票、发票、官网订单截图）
```

购物凭证是**高转化加分项**——打消"会不会是假货 / 是不是翻修"的疑虑。

## 上传流程

```
for path_or_url in images:
    result = mcp__goofish__media_upload(path_or_url)
    uploaded.append({
        "url": result["url"],
        "width": result["width"],
        "height": result["height"],
    })

# 首图 1:1 校验
first = uploaded[0]
if first["width"] != first["height"]:
    提示用户: "首图不是正方形（{w}x{h}），建议换一张 1:1 的"
```

## 失败处理

- `media_upload` 返回错误 → 多半是图片格式 / 大小问题
  - 太大：> 10MB 会失败，提示压缩
  - 格式：不支持的格式（heic 等）需要先转 jpg/png
  - 网络：重试 1 次，再失败报给用户
- 单张失败不要阻止整件发布——降级到"有几张发几张"（≥ 3 就行）

## 图片复用

用户发 10 件同款时，如果**不是完全一样**（比如颜色不同），**必须每件换首图**——否则触发"职业卖家"风控。

完全一样的货建议合并为 1 件商品，而不是发 10 次。

## 关于 AI 生成图

- **不要**给商品配 AI 生成图（闲鱼反 AI 图有简单检测，且买家反感）
- **实拍不完美**好过**AI 美图**。清晰度 > 美观度。
