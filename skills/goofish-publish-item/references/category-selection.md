# 类目选择策略

## 核心原则

**不要猜类目**。`mcp__goofish__category_recommend` 接入的是淘系类目库，比人拍脑袋准。

## 调用姿势

```
mcp__goofish__category_recommend(
    title="候选标题（尽量带品牌+核心词）",
    images=["首图URL"]
)
```

返回 `{catId, catName, confidence?}`。

## 结果处理

### Case A：单一高置信结果
直接用。

### Case B：多候选（confidence 分散）
- 展示前 3 个给用户选
- 简述每个类目的差异（比如"数码-手机 vs 数码-手机配件"）
- 不要默认选第一个

### Case C：返回 catName 与用户描述不符
例：用户说"手机壳"，API 返回"手机"
- 告诉用户这个结果
- 问"是 XX 类目吗？如果不是，能描述一下这个东西具体是什么？"
- 用户给更多细节后重调

## 常见类目踩坑

### 虚拟商品
- 游戏账号、账号代充、学习课程、代下载资源 → 必须放"虚拟类目"
- 放实物类目 → 降权 + 可能下架
- 闲鱼对虚拟商品审核严，部分子类目要实名

### 手机 vs 手机配件
- 整机（iPhone 15、小米 14） → **手机**
- 手机壳、充电器、数据线、贴膜 → **手机配件**
- 别看起来像手机就放"手机"类目

### 服饰：新 vs 二手
闲鱼默认"二手为主"。全新吊牌未剪的衣服放错子类目会触发"疑似职业卖家"审核。

### 数码：品牌子类
很多品牌下有自己的子类目（Apple → Apple 手机 / Apple 配件）。优先精细到品牌级子类目，搜索权重更高。

### 潮玩 / 手办
- 盲盒、手办、漫画周边 → "潮玩"
- 日漫周边放"二次元" vs "潮玩" 两个类目权重不同，API 给什么选什么

## 类目错放的代价

| 情节 | 后果 |
|---|---|
| 明显错（手机壳放"手机"） | 立即下架 + 类目错放记录 |
| 相近错（日系潮玩放"模型"而不是"潮玩"） | 权重 -30%，搜索排名掉 |
| 虚假类目（用"家居"发虚拟商品） | 下架 + 账户违规标记 |

## 当 API 给的类目"难用"

偶尔 `category_recommend` 会给一个偏冷门的精确类目，搜这个类目的人少。这时候：
- **还是信 API**——错类目的代价 > 冷类目的流量损失
- 如果卖家坚持要"热门"类目，提示他"可能触发类目错放，风险自担"
- 真的想要曝光，改标题优化而不是改类目

## 类目不要改来改去

- 发布后改类目 = 相当于重新审核 + 流量池重置
- 初始 24 小时内尽量不动

## 本模块不调其它工具

只调 `category_recommend`。不需要 `item_get` 也不需要反查——API 给的 catId 直接传给 `item_publish` 就够。
